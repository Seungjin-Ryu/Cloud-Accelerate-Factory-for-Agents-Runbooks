# How to Use Runbooks

> **Purpose:** This guide explains how the `01-runbooks/` folder is organized,
> how to navigate the phase-based delivery guides, and who should use each file.  
> **Audience:** CSA, CFTL, Vendor Engineer, Vendor Architect, Project Coordinator  
> **Related folders:** `02-scenarios/` (scenario-specific runbooks) · `03-patterns/`
> (reusable technical patterns)

---

## Purpose of This Runbook

The CAF Runbooks provide **structured, step-by-step delivery guidance** for executing
Cloud Accelerate Factory (CAF) for Agents engagements from start to finish.

They are designed to:

- Give **CSAs and CFTLs** a clear framework for overseeing each phase of delivery
- Give **Vendor Engineers and Architects** actionable task lists and checklists
  to execute agent builds consistently and at scale
- Ensure **repeatable, high-quality delivery** across all CAF engagements —
  regardless of region, scenario, or team
- Codify **Microsoft best practices** for agent delivery using Copilot Studio
  and/or Azure AI Foundry into a vendor-executable format
- Serve as the **single source of truth** for what needs to happen, who is
  responsible, and what must be completed before moving to the next phase

> 💡 **How CAF Runbooks work:** CAF delivery follows a **3-phase engagement model** after the Nomination & Validation phase
> — Prerequisites & Scoping → Execution & Validation → Customer Testing & Production
> Cutover. Each phase has its own Delivery Guide and Checklist. Always complete
> the checklist and confirm exit criteria before advancing to the next phase.

### Phase 1 — Prerequisites & Scoping

> **Duration:** 1–2 weeks
> **Owner:** CFTL + Vendor Project Coordinator

This is the preparation and scope definition phase before project execution begins.
The goal is to confirm all technical, licensing, and organizational prerequisites
are in place — and to produce an approved execution plan before any build work starts.

| File | Description |
|------|-------------|
| `0. Resources/` | Reference materials specific to Phase 1 (eligibility criteria, nomination portal links, scoping templates) |
| `1. Delivery-Guide.md` | Step-by-step execution guide for CSAs and Vendor teams covering environment access verification, license confirmation, stakeholder alignment, and technical scoping |
| `2. Checklist.md` | Completion checklist for Phase 1 — all items must be checked before moving to Phase 2 |

---

### Phase 2 — Execution & Validation

> **Duration:** 2–4 weeks (repeatable weekly wave model)
> **Owner:** Vendor Engineer + Vendor Architect, supported by CFTL

This is the core delivery phase where the agent solution is built, integrated,
and validated in the customer's environment. Delivery follows a weekly wave model,
with each wave producing a testable increment.

| File | Description |
|------|-------------|
| `0. Resources/` | Reference materials specific to Phase 2 (IaC templates, architecture patterns, test case templates) |
| `1. Delivery-Guide.md` | Delivery guide covering agent build (Copilot Studio / Foundry), integration setup, governance & security configuration, internal validation, and weekly checkpoint execution |
| `2. Checklist.md` | Completion criteria checklist for Phase 2 — including build, integration, governance, and internal validation sign-off before handover to customer |

---

### Phase 3 — Customer Testing & Production Cutover

> **Duration:** 1–2 weeks
> **Owner:** Customer POC (primary), supported by CFTL + Vendor Architect

This is the final phase where the customer executes UAT, the Factory team supports
defect resolution, and the agent is deployed to production. The engagement closes
with a full handover package delivered to the customer.

| File | Description |
|------|-------------|
| `0. Resources/` | Reference materials specific to Phase 3 (UAT guidance, handover templates, post-deployment guide templates) |
| `1. Delivery-Guide.md` | Execution guide covering UAT support, customer training delivery, change management support, production cutover execution, and handover documentation delivery |
| `2. Checklist.md` | Final checklist to complete production cutover and close the CAF engagement — includes UAT sign-off, go-live confirmation, and handover package completion |

## Entry Criteria by Phase

Each phase has strict entry criteria that **must be met** before work begins.
Do not proceed to the next phase until all exit criteria from the previous phase
are confirmed.

| Phase | Key Entry Criteria |
|---|---|
| **Phase 1** | Nomination accepted via CAF portal · MSX opportunity at Stage 2+ with committed milestone · Customer has $50K ACV or expected consumption · Key customer POC identified · Factory Agreement initiated |
| **Phase 2** | All Phase 1 checklist items complete · Execution plan approved by customer and CFTL · Environment access granted · Dev/Test environment provisioned · Weekly checkpoint cadence active |
| **Phase 3** | All Phase 2 checklist items complete · Internal test report approved by CFTL · No critical or high-severity open issues · Agent deployed in Test environment · UAT plan prepared by customer · Production environment provisioned |

> ⚠️ **Never skip phases or bypass exit criteria.** Skipping prerequisites is the
> most common cause of delivery delays, rework, and customer escalations in CAF
> engagements.

## Intended Audience

Different roles interact with the runbooks in different ways. Use the table below
to understand which files are most relevant to your role:

| Role | Primary Files | How to Use |
|---|---|---|
| **CFTL** (Cloud Factory Technical Lead) | All Delivery Guides + Checklists | Oversee each phase, review and approve checklists, escalate blockers, confirm exit criteria before phase transitions |
| **Vendor Engineer** | Phase 2 Delivery Guide + Checklist | Execute agent build, integration setup, and internal validation tasks; update project tracker after each wave |
| **Vendor Architect** | Phase 1 + Phase 2 Delivery Guides | Lead technical scoping, draft solution architecture, review build quality, deliver IaC templates and handover documentation |
| **Vendor Project Coordinator** | All Checklists | Track task completion across all phases, coordinate weekly checkpoints, maintain project tracker, flag blockers |
| **Account Team CSA** | Phase 1 Delivery Guide | Support nomination and eligibility confirmation; facilitate customer access and stakeholder alignment; provide post-go-live field support |
| **Customer POC** | Phase 3 Delivery Guide + Checklist | Lead UAT execution, training delivery, change management, and go-live approval; sign off on handover package |
