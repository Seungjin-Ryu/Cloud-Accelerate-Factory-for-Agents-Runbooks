# Phase 1: Prerequisites & Scoping — Delivery Guide

> **Phase Duration:** 1–2 weeks  
> **Owner:** Cloud Factory Technical Lead (CFTL) + Vendor Project Coordinator  
> **Audience:** CSA, Vendor Architect, Project Coordinator

---

## Objective

Ensure all technical, licensing, and organizational prerequisites are validated before execution begins.  
This phase establishes a shared understanding of the delivery scope, confirms stakeholder alignment, and produces a finalized execution plan — so that Phase 2 (Execution & Validation) can begin without blockers.

> ⚠️ **Do not proceed to Phase 2 until all Exit Criteria below are met.**

---

## Task List

### 1.1 Environment Access Verification

Confirm that the Factory team has the access necessary to begin hands-on-keyboard delivery.

- [ ] Request and verify **environment access** for Factory/Vendor resources to the customer tenant
- [ ] Confirm the customer has completed required **tenant configurations** (e.g., Copilot Studio environment provisioning, Power Platform admin settings)
- [ ] Validate that the customer environment meets **minimum technical requirements** for the target scenario:
  - Copilot Studio environment (non-default, dedicated to project)
  - Dataverse enabled and configured
  - Required data sources accessible (SharePoint, Dataverse, APIs, etc.)
- [ ] If hands-on-keyboard access is not feasible, confirm **session-sharing arrangement** as fallback
- [ ] Document access status and any pending items in the project tracker

> 💡 **Note:** If environment access cannot be granted within Phase 1, escalate immediately. Blocked access is the most common cause of phase delays.

---

### 1.2 License & Workload Confirmation

Verify that the customer holds the required licenses for the target workload.

#### Copilot Studio
- [ ] Confirm active **Copilot Studio license** (one of the following):
  - Power Platform: Copilot Studio
  - Power Platform: Copilot Studio (Prepaid Capacity Pack)
  - Power Platform: Copilot Studio (Pre-Purchase Plan - P3)
  - MW: Copilot Agents / MW: Copilot Agents LoB
  - MW: Copilot Connectors
- [ ] Verify **message pack capacity** — minimum 4 Copilot Message Packs (100k+ messages) expected from the scenario
- [ ] Confirm Copilot Studio environment is created and admin access is available

#### Microsoft Foundry (Azure AI Foundry)
- [ ] Confirm **Azure subscription** is active and accessible
- [ ] Verify **Azure AI Foundry** resource is provisioned or can be provisioned
- [ ] Confirm required Azure services are available: Azure AI Search, Azure OpenAI, Azure AI Document Intelligence (as applicable to scenario)
- [ ] Validate network access and firewall rules for Factory team

#### AI Builder
- [ ] Confirm **AI Builder credits** are available if document processing or form recognition is in scope
- [ ] Validate AI Builder environment settings and capacity

> 📋 **Reference:** [Supported Workloads & Eligibility — Agent-CAF.aspx](https://microsoft.sharepoint.com/sites/ABSCSU/SitePages/Agent-CAF.aspx#eligibility)

---

### 1.3 Stakeholder Alignment

Establish clear ownership and communication cadence for the engagement.

#### Executive Sponsor Confirmation
- [ ] Identify and confirm the **Executive Sponsor** on the customer side (required for project governance)
- [ ] Obtain sponsor acknowledgment of:
  - Project scope and timeline
  - Customer responsibilities during the engagement (project coordination, UAT, change management)
  - Factory's delivery model (vendor-driven, Microsoft-invested)
- [ ] Confirm sponsor availability for **kickoff meeting** and **milestone checkpoints**

#### Establish Weekly Checkpoint Cadence
- [ ] Schedule recurring **weekly checkpoint meetings** for the duration of the engagement
  - Attendees: CFTL, Vendor PM, Customer POC, Account Team CSA
  - Agenda: Progress review, blockers, next week plan
- [ ] Send calendar invites to all stakeholders before Phase 1 closes
- [ ] Agree on the **communication channel** (e.g., Teams channel, email distribution)
- [ ] Define **escalation contact** on both Microsoft and customer sides

> 👥 **Customer Responsibilities (per CAF engagement model):**  
> Customers are responsible for overall project coordination, user communications, training, UAT, and change management. The Factory team delivers the technical solution; the customer drives organizational readiness.

---

### 1.4 Technical Scoping

Define the exact scope of delivery and produce a draft architecture aligned to the target scenario.

#### Scenario Fit Assessment (In-Scope vs. Out-of-Scope)

Conduct an **initial scoping call** with the account team and customer to confirm scenario fit.

| Category | In-Scope Examples | Out-of-Scope Examples |
|---|---|---|
| General | Knowledge Retrieval (B2E), Document Processing, AI Search, LLM Fine-tuning | Major infrastructure migrations (SAP, Oracle, AIX) |
| Sales | RFP Response, Sales Qualification | Deep re-architecture or code rewrites |
| Service | Contact Center, Case Management, Knowledge Lifecycle | Custom consulting, post-delivery compliance work |
| ESS | Employee Assist, Time & Expense | VDI migrations (RDS/Citrix to AVD) |
| Finance & Supply Chain | Supplier Communications, PO Processing | Analytics platform migrations (Tableau to Power BI) |
| Governance | Agent Landing Zone | Long-term monitoring or managed services |

- [ ] Confirm the nominated scenario maps to a **pre-defined CAF pattern**
- [ ] Document any **out-of-scope items** identified during scoping — route to appropriate channel (FDE, ISD, Partner)
- [ ] Perform a **tech-agnostic assessment** to determine optimal workload: Copilot Studio vs. Azure AI Foundry vs. hybrid
- [ ] Confirm the **first sprint scope** (single use case) — full scope can be planned in parallel

#### Draft Solution Architecture
- [ ] Produce a **draft solution architecture diagram** covering:
  - User interface / entry points (Teams, Copilot, web)
  - Core agent components (Copilot Studio / Foundry)
  - Data sources and connectors (SharePoint, Dataverse, APIs, Azure AI Search)
  - Authentication and security model
  - Governance layer (Agent Landing Zone, if applicable)
- [ ] Review architecture with CFTL and Vendor Architect
- [ ] Share architecture draft with customer for feedback
- [ ] Document any open technical questions or assumptions

> 🏗️ **Reference architecture patterns** are available in [`03-patterns/`](../../03-patterns/)

## Exit Criteria (Entry Requirements for Phase 2)

All of the following must be confirmed before moving to **Phase 2: Execution & Validation**:

- [ ] ✅ Environment access granted and verified for Factory/Vendor team
- [ ] ✅ All required licenses confirmed and capacity validated
- [ ] ✅ Executive Sponsor identified and aligned on scope, timeline, and responsibilities
- [ ] ✅ Weekly checkpoint cadence established with all stakeholders
- [ ] ✅ Scenario confirmed as in-scope; out-of-scope items documented and routed
- [ ] ✅ Tech-agnostic assessment completed; target workload (Copilot Studio / Foundry) confirmed
- [ ] ✅ Draft solution architecture reviewed and approved by CFTL
- [ ] ✅ Execution plan (project plan with dates and task list) presented and approved by customer
- [ ] ✅ Phase 1 Checklist completed (see [`Checklist.md`](./Checklist.md))

## Escalation Path

If blockers arise during Phase 1, follow the escalation path below:

| Issue Type | First Contact | Escalation |
|---|---|---|
| Environment access blocked | Vendor Project Coordinator → Customer POC | CFTL → Account Team CSA |
| License gap or eligibility issue | CFTL → Account Team CSA | CAF Program contact: [CloudAccelFactory@Microsoft.com](mailto:CloudAccelFactory@Microsoft.com) |
| Scenario out-of-scope identified | CFTL assesses fit | Route to FDE ([aka.ms/FDEIntake](https://aka.ms/FDEIntake)), ISD, or Partner |
| Customer not ready / sponsor unresponsive | Account Team CSA → Customer Executive Sponsor | Escalate to CAF Program Manager |
| Technical blocker (architecture, infra) | Vendor Architect → CFTL | CFTL → CAF Technical Lead (Minhaj Sahibzada / Regional CFTL) |
| Timeline at risk | CFTL flags in weekly checkpoint | CAF Program Manager notified |
