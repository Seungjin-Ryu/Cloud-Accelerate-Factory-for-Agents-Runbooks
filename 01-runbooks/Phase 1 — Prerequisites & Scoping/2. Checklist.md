# Phase 1: Prerequisites & Scoping — Checklist

> **Phase Duration:** 1–2 weeks  
> **Owner:** Cloud Factory Technical Lead (CFTL) + Vendor Project Coordinator  
> **How to use:** Work through each section in order. All items must be checked (✅) before
> submitting for Phase 2 entry. If any item is blocked, log it in the escalation tracker.

---

## 📋 Section 1 — Customer Readiness

### 1.1 Eligibility & Commitment

- [ ] Nomination accepted via [CAF Nomination Portal](https://aka.ms/FactoryNom)
- [ ] MSX opportunity confirmed at **Sales Stage 2 or later**
- [ ] Eligible workload milestone exists and is marked as **committed** in MSX
- [ ] **$50K ACV** confirmed in Copilot Studio / Microsoft Foundry — OR — scenario expected to drive **$50K+ consumption**
- [ ] Customer is **not in pre-sales or early discovery** stage (pre-sales → route to [CAT Program](https://aka.ms/CATProgramNominate))
- [ ] Customer is in the **managed segment** (Enterprise or SMC Managed — SMB excluded)

### 1.2 Sponsorship & Access

- [ ] **Executive Sponsor** identified and confirmed on the customer side
- [ ] Executive Sponsor has acknowledged scope, timeline, and customer responsibilities
- [ ] Customer has designated a **key POC (Point of Contact)** available throughout the engagement
- [ ] Customer has agreed to provide **environment access** to Factory / Vendor resources
- [ ] [Factory Agreement](https://aka.ms/FactoryAgreement) process initiated by the account team (required if customer is non-Unified)
- [ ] Customer POC is aware of their responsibilities:
  - [ ] Overall project coordination
  - [ ] User communications & change management
  - [ ] Training delivery
  - [ ] User Acceptance Testing (UAT) lead

---

## 🔧 Section 2 — Technical Readiness

### 2.1 Environment Access Verification

- [ ] Factory / Vendor team has been granted **tenant-level access** to the customer environment
- [ ] Customer environment is **non-default**, dedicated to this project (Copilot Studio best practice)
- [ ] **Dataverse** is enabled and configured in the target environment
- [ ] Required **data sources are accessible** from the customer environment:
  - [ ] SharePoint / OneDrive
  - [ ] Dataverse tables / APIs
  - [ ] Azure AI Search index (if applicable)
  - [ ] Third-party systems / connectors (if applicable)
- [ ] **Authentication model** confirmed (Azure AD / Entra ID SSO, service accounts, etc.)
- [ ] Firewall / network rules allow Factory team access (for Azure Foundry scenarios)
- [ ] If hands-on-keyboard access is **not feasible**, session-sharing arrangement confirmed as fallback

### 2.2 License & Workload Confirmation

#### Copilot Studio
- [ ] Active Copilot Studio license confirmed (one of the following):
  - [ ] Power Platform: Copilot Studio
  - [ ] Power Platform: Copilot Studio (Prepaid Capacity Pack)
  - [ ] Power Platform: Copilot Studio (Pre-Purchase Plan - P3)
  - [ ] MW: Copilot Agents / MW: Copilot Agents LoB / MW: Copilot Connectors
  - [ ] D365 1P (with Copilot Studio milestone)
- [ ] Minimum **4 Copilot Message Packs (100K+ messages)** expected from the scenario
- [ ] Copilot Studio environment created and **admin access available**

#### Microsoft Foundry (if applicable)
- [ ] Azure subscription active and accessible
- [ ] Azure AI Foundry resource provisioned (or can be provisioned during Phase 1)
- [ ] Required Azure services confirmed available:
  - [ ] Azure OpenAI
  - [ ] Azure AI Search
  - [ ] Azure AI Document Intelligence
  - [ ] Azure Functions / Container Apps (for deployment)
- [ ] Network access and firewall rules validated for Factory team

#### AI Builder (if applicable)
- [ ] AI Builder credits available and sufficient for the scenario
- [ ] AI Builder environment settings validated

### 2.3 Scenario Fit Assessment

- [ ] Initial **scoping call** completed with account team and customer
- [ ] Nominated scenario confirmed as a **pre-defined, repeatable CAF pattern** (see [Supported Scenarios](https://microsoft.sharepoint.com/sites/ABSCSU/SitePages/Agent-CAF.aspx#technical-and-business-scenarios))
- [ ] **Tech-agnostic assessment** completed — target workload confirmed:
  - [ ] Copilot Studio
  - [ ] Azure AI Foundry
  - [ ] Hybrid (Copilot Studio + Foundry)
- [ ] Out-of-scope items identified and documented
- [ ] Out-of-scope items routed to the appropriate channel:
  - [ ] FDE → [aka.ms/FDEIntake](https://aka.ms/FDEIntake)
  - [ ] ISD / Partner engagement
  - [ ] CAT Program → [aka.ms/CATProgramNominate](https://aka.ms/CATProgramNominate)
- [ ] **First sprint scope** defined (single use case confirmed for Phase 2)

### 2.4 Architecture Baseline

- [ ] **Draft solution architecture** produced by Vendor Architect and reviewed by CFTL, covering:
  - [ ] User entry points (Teams, Copilot, web portal, etc.)
  - [ ] Core agent components (Copilot Studio topics / Foundry workflows)
  - [ ] Data sources and connectors
  - [ ] Authentication and identity model
  - [ ] Governance and security layer (Agent Landing Zone, if required)
- [ ] Architecture reviewed and **approved by CFTL**
- [ ] Architecture shared with customer and **customer feedback incorporated**
- [ ] Open technical questions and assumptions **documented** in project tracker
- [ ] Reference architecture pattern identified → see [../../03-patterns/](../../03-patterns/)

## 📅 Section 3 — Project Setup

### 3.1 Kickoff Meeting

- [ ] **Kickoff meeting** scheduled and completed with all stakeholders:
  - [ ] CFTL (Cloud Factory Technical Lead)
  - [ ] Vendor Project Coordinator and Architect
  - [ ] Customer Executive Sponsor and POC
  - [ ] Account Team CSA
- [ ] Kickoff agenda covered:
  - [ ] CAF engagement model and Factory delivery approach explained
  - [ ] Scope, timeline, and phase overview presented
  - [ ] Customer responsibilities confirmed (project coordination, UAT, change management)
  - [ ] Escalation path and contacts shared

### 3.2 Weekly Checkpoint Cadence

- [ ] Recurring **weekly checkpoint meeting** scheduled for the full engagement duration
- [ ] Calendar invites sent to all stakeholders
- [ ] Standard weekly agenda agreed:
  - Progress review against task list
  - Blockers and risks discussion
  - Next week's plan confirmed
- [ ] Meeting notes / action item tracker shared with all attendees after each session

### 3.3 Communication Channel

- [ ] Dedicated **Teams channel or group chat** created for the engagement
- [ ] All stakeholders (CFTL, Vendor team, Customer POC, Account Team) added to the channel
- [ ] **Project tracker** (task list with owners and due dates) shared and accessible to all
- [ ] Escalation contacts documented and shared:
  - CFTL name and contact
  - CAF Central: [CloudAccelFactory@Microsoft.com](mailto:CloudAccelFactory@Microsoft.com)
  - Nomination Dashboard: [aka.ms/Factory-Dashboard](https://aka.ms/Factory-Dashboard)
