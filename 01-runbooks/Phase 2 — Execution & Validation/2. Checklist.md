# Phase 2: Execution & Validation — Checklist

> **Phase Duration:** 2–4 weeks (repeatable weekly wave model)
> **Owner:** Vendor Engineer + Vendor Architect, supported by CFTL
> **How to use:** Complete each section in order, wave by wave. All items must be checked (✅)
> before submitting for Phase 3 entry. Blockers must be logged in the project tracker immediately.

---

## 🏗️ Section 1 — Agent Build

### 1.1 Copilot Studio Agent Build

- [ ] Dedicated **Copilot Studio environment** created (non-default, project-specific)
- [ ] Agent **topics and conversation flows** configured to cover all target scenarios
- [ ] **Trigger phrases** defined and tested for each topic
- [ ] **Generative Answers** or GPT-based fallback configured and grounding verified
- [ ] **System prompt / LLM output steering** applied (tone, style, language constraints)
- [ ] **Authentication** configured within Copilot Studio:
  - [ ] Azure AD / Entra ID SSO — OR —
  - [ ] Manual authentication / Anonymous (with CFTL approval)
- [ ] Agent **publishing channel** configured:
  - [ ] Microsoft Teams
  - [ ] Microsoft 365 Copilot
  - [ ] Web chat / Custom channel
- [ ] Agent **registered with Agent 365** ✅ *(required for all CAF deliveries)*

### 1.2 Azure AI Foundry Build *(if applicable)*

- [ ] Azure AI Foundry resources provisioned:
  - [ ] Azure OpenAI (model deployment confirmed)
  - [ ] Azure AI Search (index created and populated)
  - [ ] Azure AI Document Intelligence (if document processing is in scope)
- [ ] **RAG pipeline** implemented and validated:
  - [ ] Chunking strategy defined and applied
  - [ ] Embedding model selected and configured
  - [ ] Index refresh schedule confirmed with customer
- [ ] Application deployed on **Azure Functions or Azure Container Apps**
- [ ] **Architecture type** confirmed and applied:
  - [ ] Standard Architecture *(scrubbed / non-confidential data)*
  - [ ] ZTA Architecture *(confidential / production data)*
- [ ] **IaC templates** (Bicep / Terraform) used for environment configuration
- [ ] **Key Vault and Managed Identity** integrations configured

---

## 🔌 Section 2 — Connectors, APIs & Data Sources

### 2.1 Connectors

- [ ] All **Power Platform connectors** (standard or custom) configured and authenticated
- [ ] **Microsoft Graph connectors** set up for SharePoint / Teams / OneDrive (if applicable)
- [ ] **Dataverse connectors** configured for CRM or ERP data (if applicable)
- [ ] Connector **permissions scoped to minimum required access** (least-privilege validated)
- [ ] Each connector tested **end-to-end in Dev environment** before promotion to Test

### 2.2 APIs

- [ ] All **external APIs** required by the scenario identified and documented
- [ ] **Power Automate flows** created to invoke APIs from Copilot Studio
- [ ] API **authentication** configured and validated:
  - [ ] OAuth 2.0 — OR — API key — OR — Managed Identity
- [ ] **API response handling and error management** implemented within agent flow
- [ ] All API endpoints, authentication methods, and data contracts **documented**

### 2.3 Data Sources

- [ ] All required data sources confirmed accessible from the agent environment:
  - [ ] SharePoint / OneDrive
  - [ ] Dataverse
  - [ ] Azure Blob Storage
  - [ ] SQL / PostgreSQL / Cosmos DB *(if applicable)*
  - [ ] Microsoft Fabric / OneLake *(if applicable)*
- [ ] **Data classification** confirmed for each source:
  - [ ] Public / Internal → Standard Architecture
  - [ ] Confidential → ZTA Architecture
- [ ] **AI Search index** validated:
  - [ ] Data ingested and indexed correctly
  - [ ] Index refresh schedule confirmed
  - [ ] Search relevance tested with representative queries

---

## 🔒 Section 3 — Governance & Security

### 3.1 Agent Landing Zone *(if in scope)*

- [ ] **Power Platform DLP policies** configured and applied to the project environment
- [ ] **Environment-level security groups** and role assignments set up
- [ ] **Tenant-level agent management settings** configured (creator/publisher permissions)
- [ ] **Audit logging** enabled for agent interactions and connector usage
- [ ] Agent Landing Zone configuration reviewed and approved by CFTL

### 3.2 Responsible AI & Guardrails

- [ ] **Content moderation settings** configured in Copilot Studio:
  - [ ] Topic filtering enabled for sensitive or out-of-scope queries
  - [ ] Safety system prompts applied
  - [ ] Fallback behaviors defined for no-match and sensitive query scenarios
- [ ] **Conversation logging** enabled for compliance and review purposes
- [ ] Responsible AI requirements confirmed as met with customer IT Admin

### 3.3 Authentication & Identity

- [ ] **Entra ID SSO** configured and tested end-to-end *(if applicable)*
- [ ] **Service account permissions** scoped to minimum required access
- [ ] No over-permissioned accounts or shared credentials in use
- [ ] Authentication tested across all publishing channels

### 3.4 Observability & Monitoring

- [ ] **Copilot Studio analytics** enabled:
  - [ ] Session volume tracking active
  - [ ] Escalation rate monitoring configured
  - [ ] CSAT / feedback collection enabled *(if applicable)*
- [ ] **Azure Monitor / Application Insights** configured for Foundry-based agents *(if applicable)*
- [ ] **Alerting thresholds** defined for error rates and response latency
- [ ] Monitoring dashboard or report shared with customer POC

---

## ✅ Section 4 — Internal Validation

### 4.1 Development Environment Testing

- [ ] **Unit-level testing** completed for individual topics, flows, and API integrations
- [ ] **Generative AI response quality** validated:
  - [ ] Representative user queries tested across all key scenarios
  - [ ] Grounding verified (responses cite correct sources, no hallucination observed)
  - [ ] Tone, language, and format confirmed to match customer requirements
- [ ] **Authentication flows** tested end-to-end (login, token refresh, session timeout)
- [ ] **Connector and API responses** validated under normal and edge-case inputs
- [ ] All bugs and issues logged in the project tracker with **severity ratings**

### 4.2 Test Environment Validation

- [ ] Build promoted from **Dev → Test environment** successfully
- [ ] **End-to-end functional testing** completed (full conversation flow covered)
- [ ] **Multi-turn conversation** scenarios tested (context retention, slot-filling, disambiguation)
- [ ] **Fallback and error handling** behaviors validated:
  - [ ] No-match scenario
  - [ ] API timeout / data unavailable scenario
  - [ ] Out-of-scope query scenario
- [ ] **Regression testing** completed after all fixes
- [ ] **Performance baseline** established:
  - [ ] Average response latency measured and within acceptable threshold
  - [ ] Throughput tested under expected concurrent user load
- [ ] No **critical or high-severity** open issues remaining
- [ ] **Internal test report** completed and shared with CFTL

### 4.3 CFTL Sign-off

- [ ] CFTL has reviewed the **internal test report**
- [ ] CFTL has confirmed all critical and high-severity issues are resolved
- [ ] CFTL has approved the build for **handover to customer (Phase 3)**
- [ ] Agent confirmed as published to correct channel(s) in Test environment

---

## 📅 Section 5 — Weekly Checkpoint Execution

- [ ] **Week 1 checkpoint** completed — initial build increment demonstrated
- [ ] **Week 2 checkpoint** completed — integration progress reviewed
- [ ] **Week 3 checkpoint** completed — validation results reviewed *(adjust based on actual duration)*
- [ ] Meeting notes and action items distributed within **24 hours** of each checkpoint
- [ ] Project tracker updated after each checkpoint
- [ ] No unresolved **critical blockers** carried over across two consecutive checkpoints
