# Phase 3: Customer Testing & Production Cutover — Delivery Guide

> **Phase Duration:** 1–2 weeks  
> **Owner:** Customer POC (primary), supported by CFTL + Vendor Architect  
> **Audience:** Customer IT Team, End Users, CFTL, Vendor Architect, Account Team CSA

---

## Objective

Support the customer through User Acceptance Testing (UAT), deliver training and change
management guidance, execute production cutover, and hand over all documentation — so
the agent solution is **live in production, adopted by users, and independently operable**
by the customer after CAF engagement closes.

> ⚠️ **Important:** Phase 3 is primarily **customer-led**. The Factory team provides
> guidance and support, but the customer owns UAT execution, training delivery,
> change management, and production go-live decision. CAF does **not** provide
> post-delivery monitoring, compliance work, or custom consulting after handover.

---

## Entry Criteria

The following must be confirmed before Phase 3 begins:

- [ ] All Phase 2 Exit Criteria confirmed
  (see ../Phase-2-Execution-Validation/Delivery-Guide.md)
- [ ] Internal test report completed and approved by CFTL
- [ ] No critical or high-severity open issues remaining from Phase 2
- [ ] Agent deployed and accessible in the **Test environment**
- [ ] Agent registered with **Agent 365**
- [ ] UAT plan prepared by customer (test scenarios, test users, success criteria defined)
- [ ] Customer training materials drafted or provided by Factory team
- [ ] Production environment provisioned and access confirmed
- [ ] Phase 3 kickoff date agreed with all stakeholders

## Task List

### 3.1 UAT Support (User Acceptance Testing)

UAT is **led and executed by the customer**. The Factory team provides guidance,
answers technical questions, and resolves bugs identified during testing.

#### UAT Preparation
- [ ] Share **UAT guidance document** with the customer, covering:
  - Recommended test scenario coverage (happy path, edge cases, error handling)
  - How to log and categorize defects (critical, high, medium, low)
  - Success criteria definition (e.g., pass rate threshold, response quality score)
  - Evaluation framework guidance (how to assess generative AI response quality)
- [ ] Confirm **UAT test users** are identified and have access to the Test environment
- [ ] Confirm UAT **start date, duration, and completion deadline**

#### UAT Execution Support
- [ ] Factory team available during UAT period to answer **technical questions**
  (via agreed communication channel — Teams chat, email, etc.)
- [ ] Triage and resolve **defects raised by customer** during UAT:
  - Critical defects: resolved within the same day
  - High defects: resolved within 1–2 business days
  - Medium / Low defects: resolved before production cutover or documented
    as known limitations
- [ ] Conduct **regression testing** after each defect fix before re-releasing to customer
- [ ] Maintain a **UAT defect tracker** (shared with customer):
  - Issue description, severity, status, owner, resolution date
- [ ] Hold **UAT checkpoint meeting** mid-UAT to review progress and address blockers
- [ ] Confirm **UAT completion sign-off** from customer POC before proceeding to
  production cutover

> 💡 **Note:** CAF provides guidance on evaluation frameworks for generative AI quality
> (e.g., groundedness, relevance, coherence scoring), but does not conduct formal UAT
> on behalf of the customer. Full UAT ownership belongs to the customer.

---

### 3.2 Customer Training Delivery

Equip the customer's team with the knowledge and skills to operate, manage, and
expand the agent solution independently after handover.

#### Training Preparation
- [ ] Identify **training audience and personas**:
  - End users (agent consumers)
  - IT Admin / Power Platform Admin (agent operators)
  - Agent makers / citizen developers (if customer plans to extend the agent)
- [ ] Confirm **training format and delivery method**:
  - [ ] Live session (Teams meeting / on-site)
  - [ ] Recorded walkthrough (async delivery)
  - [ ] Written guide (embedded in handover documentation)
- [ ] Prepare **training materials** covering:
  - How to interact with the agent (end user guide)
  - How to manage and monitor the agent (admin guide)
  - How to update topics, knowledge sources, and connectors (maker guide)
  - How to interpret analytics and act on conversation insights

#### Training Delivery
- [ ] Deliver **end user training** session or materials
- [ ] Deliver **admin / operator training** session or materials:
  - Copilot Studio environment management
  - DLP policy management
  - Agent 365 monitoring and governance
  - Connector and integration maintenance
- [ ] Deliver **maker / developer training** (if customer intends to extend the agent):
  - Adding new topics and conversation flows
  - Updating knowledge base / AI Search index
  - Publishing new agent versions
- [ ] Confirm training completion with customer POC
- [ ] Collect training feedback from attendees

> 📋 **Note:** Training delivery is the customer's responsibility to organize and
> schedule. The Factory team provides materials and facilitates sessions upon request.
> Post-delivery training at scale is out of CAF scope.

---

### 3.3 Change Management Support

Help the customer prepare their organization for the agent's production launch —
ensuring user awareness, adoption readiness, and smooth go-live.

- [ ] Confirm customer has an **internal change management owner** identified
  (typically the customer POC or a designated change lead)
- [ ] Provide **go-live communication template** for customer to send to end users:
  - What the agent does and why it was deployed
  - How to access and use the agent
  - Where to report issues or provide feedback
  - Who to contact for support after go-live
- [ ] Confirm **feedback and support channel** is set up for post-go-live user queries
  (e.g., dedicated Teams channel, email alias, helpdesk ticket category)
- [ ] Review **adoption readiness checklist** with customer:
  - [ ] End users informed about the agent launch
  - [ ] Support escalation path communicated to users
  - [ ] Feedback collection mechanism in place
  - [ ] Agent usage metrics tracking confirmed active
- [ ] Confirm customer has a plan for **ongoing agent maintenance**:
  - Knowledge base / AI Search index refresh cadence
  - Topic update process (who owns, how often reviewed)
  - Governance review cycle (DLP, security, audit log review)

> 💡 **Note:** Change management execution (communications, training rollout,
> adoption campaigns) is the customer's responsibility. CAF provides guidance
> and templates, not full change management delivery.

---

### 3.4 Production Cutover Execution

Deploy the validated agent to the production environment and confirm go-live.

#### Pre-Cutover Validation
- [ ] Confirm all **UAT sign-off** criteria met (no critical/high open defects)
- [ ] Confirm **production environment** is provisioned and matches Test environment
  configuration (environment variables, connections, security settings)
- [ ] Confirm **production data sources** are accessible and indexed correctly
- [ ] Validate **authentication** in production environment (Entra ID SSO, service accounts)
- [ ] Confirm **DLP policies and governance settings** applied to production environment
- [ ] Conduct **final smoke test** in production before go-live announcement

#### Cutover Execution
- [ ] Promote agent build from **Test → Production environment**
- [ ] Publish agent to production **channel(s)**:
  - [ ] Microsoft Teams
  - [ ] Microsoft 365 Copilot
  - [ ] Web chat / Custom channel
- [ ] Confirm **Agent 365 registration** updated to reflect production deployment
- [ ] Validate agent is **accessible and functional** for a representative set of
  production users (pilot group or full rollout, per customer's plan)
- [ ] Confirm **monitoring and alerting** is active in production:
  - Copilot Studio analytics dashboard confirmed live
  - Azure Monitor / Application Insights active (Foundry agents)
  - Alerting thresholds validated

#### Go-Live Confirmation
- [ ] Customer POC confirms **production go-live** is approved
- [ ] CFTL confirms **Factory delivery is complete**
- [ ] Go-live announcement sent to end users by customer
- [ ] Field CSA notified for post-go-live production support
  (Field CSA can be included to support production go-live activities or address
  out-of-scope needs per CAF engagement model)

---

### 3.5 Handover Documentation Delivery

Deliver all technical assets and documentation required for the customer to operate,
maintain, and expand the agent solution independently.

#### Reference Architecture
- [ ] Finalize and deliver the **solution architecture document**, including:
  - Logical architecture diagram (updated to reflect as-built state)
  - Component descriptions (Copilot Studio, Foundry, connectors, data sources)
  - Authentication and identity model
  - Governance and security layer (DLP, Agent Landing Zone, if applicable)
  - Integration map (all connectors, APIs, data sources with endpoints documented)
- [ ] Architecture document reviewed and approved by CFTL before delivery
- [ ] Stored in the agreed **handover location** (SharePoint, Teams channel, or GitHub)

#### Infrastructure as Code (IaC) Templates
- [ ] Deliver all **Bicep / Terraform IaC templates** used during the engagement:
  - Azure AI Foundry resource provisioning templates
  - Azure AI Search index configuration scripts
  - Key Vault and Managed Identity setup templates
  - Agent Landing Zone deployment templates (if applicable)
- [ ] Templates include **inline comments** explaining each configuration parameter
- [ ] Templates tested and confirmed reproducible in a clean environment
- [ ] README file included explaining how to use each template

#### Post-Deployment Guide
- [ ] Deliver the **post-deployment operations guide**, covering:
  - **Day-to-day operations:** how to monitor agent health, review analytics,
    and respond to alerts
  - **Knowledge base maintenance:** how to update AI Search index, refresh
    data sources, and manage document ingestion
  - **Topic and flow updates:** how to add/edit topics in Copilot Studio,
    test changes, and publish updates
  - **User management:** how to add/remove users, manage access permissions,
    and update security groups
  - **Troubleshooting guide:** common issues and resolution steps
    (aligned with Phase 2 Known Issues)
  - **Escalation contacts:** who to contact for Microsoft support
    (CAF Central, Account Team CSA, support ticket process)
  - **Next steps / expansion roadmap:** suggestions for extending the agent
    to additional scenarios or user groups

#### Handover Package Summary
- [ ] All documents consolidated in a **single handover package** accessible to the customer
- [ ] Handover package contents confirmed complete with customer POC
- [ ] Customer POC signs off on receipt of handover package
