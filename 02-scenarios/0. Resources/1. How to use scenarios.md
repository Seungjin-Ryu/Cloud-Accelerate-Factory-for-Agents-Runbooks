## Scenario Folder Structure

Each scenario folder follows a standardized structure to ensure consistency, reusability, and clarity across different agent scenarios.

Below is an example based on the **HR Onboarding Agent** scenario:

| File | Description |
|------|-------------|
| `Overview.md` | Provides a high-level overview of the scenario, including the supported CAF scenario category (e.g., Employee Self-Service), customer eligibility criteria, and expected outcomes. |
| `Architecture.md` | Contains the logical architecture diagram and detailed explanations of key components and data flows. Previously generated architecture diagrams can be reused here. |
| `Runbook.md` | Scenario-specific delivery procedures for Phase 1–3, customized from the standard runbooks under the `01-runbooks` folder to fit the HR scenario. |
| `Sample_prompts.md` | Includes example conversations with the HR Onboarding Agent, along with sample edge-case handling scenarios. |
