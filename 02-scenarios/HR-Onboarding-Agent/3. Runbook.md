# Step-by-Step Deployment Guide — HR Onboarding Auto Agent

> **Scenario**: HR Self-Help Autonomous Assistant  
> **Platform**: Microsoft Copilot Studio + Power Platform + ServiceNow  
> **Target Users**: CSA / Vendor / PM  
> **Last Updated**: March 2026

---

## Overview

New hires often face challenges understanding company policies, accessing
necessary resources, and integrating into the team. This agent streamlines
the onboarding process by:

- Answering HR-related questions using a ServiceNow knowledge base
- Automatically responding to email inquiries sent to an HR mailbox
- Being accessible via Microsoft Teams and M365 Copilot

---

## Prerequisites

Before starting, confirm the following are in place:

| Requirement | Details |
|---|---|
| Microsoft 365 Copilot License | Active license required for the maker account |
| Power Platform Admin Rights | Needed to create a Developer Environment |
| ServiceNow Tenant | Required for knowledge connector setup |
| Teams Admin Access | Required to enable custom app sideloading |
| M365 Admin Access | Required to publish the agent org-wide |

---

## Phase 0: Environment Setup

### Step 0-1. Verify Copilot Extensibility is Enabled

1. Go to **Microsoft 365 Admin Center** → [https://admin.microsoft.com](https://admin.microsoft.com)
2. Navigate to **Integrated Apps** → **Available apps**
3. Click the **Settings icon** (top right)

   > 📸 **[Screenshot required]** — Integrated Apps page showing the Available apps tab and Settings icon

4. In the panel, find **"Allow the following users access to Copilot agents"**
5. Confirm it is set to **"All users"**, or verify your user is included in a specific group

   > 📸 **[Screenshot required]** — "Allow the following users access to Copilot agents" settings panel

---

### Step 0-2. Enable Custom App Sideloading in Microsoft Teams

1. Sign in to **Teams Admin Center** → [https://admin.teams.microsoft.com](https://admin.teams.microsoft.com)
2. Go to **Teams apps** → **Manage apps**
3. Click **Actions** (top right) → **Org-wide app settings**
4. Under **Custom apps**, toggle **"Let users interact with custom apps in preview"** → **On** → Click **Save**

   > 📸 **[Screenshot required]** — Org-wide app settings panel showing the Custom apps toggle

5. Go to **Teams apps** → **Setup policies** → **Global (Org-wide default)**
6. Toggle **"Upload custom apps"** → **On** → Click **Save**

---

### Step 0-3. Create Power Platform Developer Environment

1. Go to **Power Platform Admin Center** → [https://admin.powerplatform.microsoft.com](https://admin.powerplatform.microsoft.com)
2. Click **Environments** → **+ New**

   > 📸 **[Screenshot required]** — Power Platform Admin Center, Environments page with the +New button

3. Fill in the following values:

   | Field | Value |
   |---|---|
   | Name | `Copilot` (or any preferred name) |
   | Region | Select the region closest to your tenant |
   | Type | **Developer** |
   | Managed Environment | No |

   > 📸 **[Screenshot required]** — New Environment settings panel showing Name, Region, and Type fields

4. Click **Next** → **Next** → **Save**

### Step 1-2. Add HR Documents to ServiceNow Knowledge Base

> **Purpose**: The out-of-the-box ServiceNow tenant only includes IT-related documents. We need to add HR-specific content for the onboarding agent.

#### Article 1: Contoso Employee Handbook

1. Visit your ServiceNow portal → `https://[your-instance].service-now.com`
2. Log in with your ServiceNow credentials

   > 📸 **[Screenshot required]** — ServiceNow login screen

3. Search **"knowledge"** in the top search bar
4. Navigate to **Knowledge** → **Administration** → **Knowledge Bases**

   > 📸 **[Screenshot required]** — ServiceNow left navigation showing Knowledge > Administration > Knowledge Bases

5. Click **"IT"** to open the IT Knowledge Base

   > 📸 **[Screenshot required]** — Knowledge Bases list with "IT" selected

6. **Change the Publish Workflow to Instant Publish**:
   - Click the magnifier icon next to **"Publish workflow"**
   - In the dialog, select **"Knowledge – Instant Publish"**
   - Click **Update** to save

   > 📸 **[Screenshot required]** — Workflow change dialog with "Knowledge – Instant Publish" selected

7. Return to the **IT Knowledge Base** page
8. In the **Knowledge section**, click **"New"**

   > 📸 **[Screenshot required]** — IT Knowledge Base page showing the "New" button in the Knowledge section

9. On the **"Select the Article Template"** page, click **"Standard"**

   > 📸 **[Screenshot required]** — Article Template Selector page with Standard selected

10. Fill in the following fields:

    | Field | Value |
    |---|---|
    | Short description | `Contoso Employee Handbook` |
    | Article body | Paste content from `Contoso Employee Handbook.docx` → Choose **"Keep formatting"** when prompted |
    | Category | Click the magnifier icon → Click **"+"** → Type `Policy` → Click **OK** |

    > 📸 **[Screenshot required]** — Knowledge New Record page showing Short description, Article body, and Category fields

11. Click **Submit**
12. Find the newly created article (Status: "Draft") → Click the **article Number** link
13. Click **"Publish"** to publish the article

    > 📸 **[Screenshot required]** — Knowledge edit page with the "Publish" button highlighted

14. Confirm the article's **Workflow status** shows **"Published"**

    > 📸 **[Screenshot required]** — Article showing "Published" status in the Workflow column

## Phase 2: Agent Creation in Copilot Studio

### Step 2-1. Create Custom Agent

1. Go to **Copilot Studio** → [https://copilotstudio.microsoft.com](https://copilotstudio.microsoft.com)
2. Select the **Developer Environment** created in Phase 0

   > 📸 **[Screenshot required]** — Copilot Studio environment selection screen

3. Click **"Agents"** in the left navigation

4. Click **"New agent"**

   > 📸 **[Screenshot required]** — Agents page with the "New agent" button

5. Click **"Configure"** and fill in the following:

   |---|---|

   | Description | `HR Onboarding Agent` |

6. Click **"Create"**

   > 📸 **[Screenshot required]** — Configure page with Name and Description filled in, and the Create button

---

### Step 2-2. Add ServiceNow Knowledge

1. On the Agent Overview page, click **"Add knowledge"**

   > 📸 **[Screenshot required]** — Agent Overview page with the "Add knowledge" button

2. In the **"Add knowledge"** dialog, click **"Featured"**
3. Select **"ServiceNow"**

   > 📸 **[Screenshot required]** — Add knowledge dialog with ServiceNow option selected

4. In the **"Select ServiceNow connection"** dialog, select your ServiceNow connector (e.g., `ServiceNowKB5`)
5. Click **"Add"**

   > 📸 **[Screenshot required]** — ServiceNow connector selection and Add button

6. After adding, confirm the knowledge source appears on the Overview page
7. Locate the **"Web Search"** toggle → Click to **disable** it

   > ⚠️ **Important**: Disabling Web Search ensures the agent only uses the ServiceNow knowledge base and does not draw from the LLM's general knowledge.

   > 📸 **[Screenshot required]** — Web Search toggle in the disabled state

---

### Step 2-3. Add SendEmail Tool

1. Click **"Add tool"**

   > 📸 **[Screenshot required]** — Agent Overview page with the "Add tool" button

2. In the **"Add tool"** dialog, click **"Office 365 Outlook"**  
   *(If not visible, search for "Office 365 Outlook")*

   > 📸 **[Screenshot required]** — Add tool dialog with Office 365 Outlook selected

3. Click **"Send an email (V2)"**

   > 📸 **[Screenshot required]** — "Send an email (V2)" option selected

4. Confirm the connection uses the **current maker's credentials**
5. Click **"Add and configure"**

   > 📸 **[Screenshot required]** — Credentials confirmation and "Add and configure" button

6. Update the **Name** field to: `SendEmail`
7. Leave all other fields as default
8. Click **"Save"**

   > 📸 **[Screenshot required]** — Tool name set to "SendEmail" with the Save button

9. Click the **"Overview"** tab to return to the main agent page

---

### Step 2-4. Add Email Trigger

1. On the **Overview** page, click **"Add trigger"**

   > 📸 **[Screenshot required]** — Overview page with the "Add trigger" button

2. In the **"Add trigger"** dialog, select **"When a new email arrives (V3)"**
3. Click **"Next"**

   > 📸 **[Screenshot required]** — Add trigger dialog with "When a new email arrives (V3)" selected

4. Keep the default **Trigger name**
5. Confirm connections for both:
   - **Microsoft Copilot Studio** — current maker's credentials
   - **Office 365 Outlook** — current maker's credentials
6. Click **"Next"**

   > 📸 **[Screenshot required]** — Trigger connection credentials configuration screen

7. Keep the default configuration in the **"When a new email arrives"** dialog
8. Click **"Create trigger"**

   > 📸 **[Screenshot required]** — "Create trigger" button clicked

9. Click **"Close"** to dismiss the confirmation dialog

### Step 2-5. Update Agent Instructions

1. On the Overview page, click **"Edit"** next to **Instructions**
2. Replace the instructions with the following content:

   > ⚠️ Replace `[ServiceNowKB]` with your actual knowledge name (e.g., `ServiceNowKB5`)  
   > ⚠️ Replace `[SendEmail]` with your tool name. Use `/` to select a Tool inline.

# HR Onboarding Autonomous Agent

## Prerequisites

### Licensing & Permissions
- Microsoft 365 Copilot license
- Copilot extensibility enabled in tenant
- Permission to create Power Platform environments
- Permission to configure Copilot connectors
- Teams custom app side-loading enabled

### Environment
- Dedicated Power Platform **Developer Environment**
- Dataverse enabled
- Non-default environment recommended

---

## Knowledge Preparation (ServiceNow)

### ServiceNow Copilot Connector
- Create ServiceNow developer tenant
- Configure ServiceNow Copilot Connector
- Verify connector availability in Microsoft 365 Search & Intelligence

### HR Knowledge Content
Create and publish the following articles in ServiceNow Knowledge Base:
- **Employee Handbook** (Category: Policy)
- **Health & Wellness Benefits** (Category: Benefit)

### Indexing
- Run **Full Crawl** on ServiceNow Copilot Connector
- Confirm indexed item count increases before agent testing

---

## Agent Build & Configuration

### Create Custom Agent
- **Name**: `HROnboardingAutoAgent`
- **Description**: HR Onboarding Agent
- **Orchestration**: Generative (default)

### Add Knowledge Source
- Add ServiceNow Copilot Connector
- Disable Web Search to enforce authoritative grounding

### Add Tools
- Tool: **Office 365 Outlook – Send an email (V2)**
- Tool Name: `SendEmail`
- Authentication: Maker credentials

### Configure Trigger
- Trigger: **When a new email arrives (V3)**
- Logic:
  - Extract question from email body
  - Search ServiceNow knowledge
  - Respond to sender via email

---

## Agent Instructions

```text
Capabilities:
This is an HR onboarding agent for new hires.

HR Questions:
- Use ServiceNow knowledge only.
- If no result is found, respond with:
  "I don't know, please reach out to HR."

Email Trigger Logic:
- Question is stored in email body.
- Search ServiceNow knowledge.
- Send response email to sender.
- Email body must be HTML formatted.
```

## Suggested Starter Prompts

1. What is wellness benefit program (Benefits)

3. What is the travel expense?(Travel Expense)
